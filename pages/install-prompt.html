## SW 디자인 시스템 V2.4 — 토큰 컨텍스트

아래 CSS 토큰은 SW 디자인 시스템 V2.4의 공식 정의입니다.
이 토큰만 사용하여 UI를 구현해주세요.

---

### 핵심 규칙

1. **색상은 반드시 Semantic 토큰 경유**
   - Component Token → Semantic Token → Foundation Foundation
   - HEX 값 직접 사용 금지 (예외: rgba overlay, dark border)

2. **크기·간격은 Semantic 또는 Foundation 참조**
   - 여러 컴포넌트가 공유하는 값 → Semantic 경유
   - 컴포넌트 전용 값 → Foundation 직접 참조 가능

3. **Light/Dark 테마**
   - :root = Light (기본)
   - [data-theme="dark"] = Dark
   - 변수명은 동일, 값만 변경

4. **컴포넌트 토큰은 Semantic 참조**
   - --button-primary-default-bg: var(--color-action-primary-default)
   - Foundation Foundation를 Component에서 직접 참조하지 않음
