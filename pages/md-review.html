--color-bg-home 처리 방식 선택
Figma에서 #F5F6FB (미세한 Blue-tinted gray)를 직접 하드코딩 중.
기존 Primitive와 값 차이가 작아 아래 두 방향 중 하나를 확정해야 합니다.
A 권장 — Foundation에 --color-blue-gray-0: #F5F6FB 신규 Primitive 추가.
홈 배경의 독립적인 의미를 보존하고, 향후 다른 값으로 교체 가능.
B --color-bg-subtle: #F5F5F5로 통합. 시각적 차이(4 hex)가 미미해 실사용에서 구분 불가능한 경우 단순화.

💡 수정 제안 · 설계 근거 참고용 — 긴급 아님
다크모드 border에 rgba 사용하는 이유
다크 테마에서 color-border-*를 고정 hex 대신 rgba(255,255,255,0.04 ~ 0.20)으로 정의했습니다.
고정 hex를 쓰면 배경 명도가 다른 여러 영역(카드 위, 모달 위 등)에서 테두리 시인성이 달라집니다.
rgba를 쓰면 어떤 배경 위에서도 일정한 레이어 깊이를 표현할 수 있어 다크 UI 일관성이 높아집니다.
--color-text-correct vs --color-text-link 분리 유지 이유
현재 두 토큰 모두 Light: #1D6CEB, Dark: #6FA5F8로 동일합니다.
correct는 ID·PW 올바른 입력 피드백 전용이고, link는 하이퍼링크·액센트 텍스트 전용입니다.
향후 올바른 입력 피드백 색상을 독립적으로 조정할 때를 대비해 분리 유지를 권장합니다.
--color-text-helper vs --color-text-placeholder 분리 유지 이유
현재 두 토큰 모두 Light: #9D9D9D, Dark: #55575F로 동일합니다.
placeholder는 인풋 필드 안의 힌트 텍스트, helper는 필드 아래 도움말 텍스트입니다.
일부 서비스에서 helper 텍스트에 다른 색상을 적용하는 사례가 있어 분리 유지를 권장합니다.
border/disabled-alt1 vs alt2 처리 방식
컴포넌트 유형별로 disabled 테두리 색상이 다릅니다 (Input: gray/200, Select: gray/50).
Semantic 레이어에서는 단일 --color-border-default를 제공하고,
Component Token 레이어에서 각각 override합니다:
  --input-disabled-border: var(--color-border-default) (alt1)
  --select-disabled-border: var(--color-border-subtle) (alt2)
