Legacy란?
이 디자인시스템에서 "Legacy"의 정의와 운영 원칙
📦
Legacy는 S1 UX Design Guide 2.4 Figma 파일에 정의된 구버전 Variable/Style을 가리킵니다. 신규 디자인시스템(v0.1)의 Semantic / Component 토큰으로 전환 중인 항목입니다. Legacy 토큰은 기존 서비스 유지용으로만 허용하며, 신규 화면에서는 사용하지 않습니다.
분류설명신규 사용
confirmed canonical 매핑이 확정되어 교체 가능한 항목 신규 토큰으로 교체 권장
needs-review canonical 후보가 있으나 사람 검토 필요 항목 보류 — 검토 완료 후 교체
no-usage-observed MCP 스캔에서 사용 미확인 항목 MVP-L1 완료 후 재판단
마이그레이션 접근 방법
단계적 전환 전략과 도구
  1. 1
    Source Guard로 현황 파악
    npm run guard -- --target ../service-project를 실행하면 서비스 프로젝트 내 HEX 직접 사용, Foundation 직접 참조, Legacy 토큰 사용 등을 자동으로 검출합니다.
  2. 2
    confirmed 항목부터 교체
    아래 "확정 매핑 테이블"의 confirmed 항목은 즉시 교체 가능합니다. Legacy 변수명을 canonical CSS 변수로 1:1 교체합니다.
  3. 3
    needs-review 항목 개별 검토
    canonical 후보가 있으나 불확실한 항목은 pages/md-review.html에 검토 항목으로 등록 후 결정합니다. 임의로 교체하지 않습니다.
  4. 4
    MVP-L1 완료 후 전수 감사
    Figma Plugin → Export Variables → npm run figma:audit를 실행하면 Legacy Variable 전체 목록을 canonical 토큰과 자동 매핑합니다.
# Source Guard 사용 예시 npm run guard -- --target ../my-service # 검사 리포트 생성 npm run guard:suggest -- --target ../my-service # 수정 후보 생성 npm run guard:apply:dry -- --target ../my-service # dry-run (파일 수정 없음) npm run guard:apply -- --target ../my-service # high-confidence 자동 적용

#항목도구완료 기준
1 Source Guard 검사 실행 — Legacy 토큰 및 HEX 위반 현황 파악 npm run guard reports/source-guard-{target}.md 생성됨
2 confirmed 항목 일괄 교체 npm run guard:apply:dry high-confidence 자동 적용 완료 (backup 생성 확인)
3 needs-review 항목 개별 검토 + MD Review 등록 md-review.html 결정 항목 모두 confirmed 또는 deprecated 처리됨
4 Dark Mode 토큰 전환 검증 semantic.html 테마 전환 탭 data-theme="dark"에서 시각 이상 없음
5 Source Guard 재실행 — 위반 0건 확인 npm run guard error 0건, warning 수용 가능 수준
