다운로드
프로젝트 마무리 단계

① 문구 검토하기

프로젝트 마무리 단계에서 작성된 UX Writing을 최종 검수할 때 사용하는 기능입니다. 선택한 텍스트를 네이버 맞춤법 API와 사내 UX Writing 용어집을 기반으로 검토하며, 맞춤법뿐 아니라 용어 통일·권장 문구·표기 규칙 등 약 149개의 검토 규칙을 적용해 서비스 가이드에 맞는 문구인지 확인할 수 있습니다.

문구 검토하기네이버 맞춤법 + 149개 규칙 · 실제 동작
입력한 문구가 네이버·검토 서버로 전송됩니다
검토하기를 누르면 여기에 결과가 표시됩니다
프로젝트 초기 단계

② 문구 추천받기

프로젝트 초기 단계에서 버튼·안내문·오류 메시지 등 어떤 문구를 사용할지 고민될 때 사용하는 기능입니다. 입력 또는 선택한 텍스트를 Claude가 분석하여 서로 다른 접근의 추천 문구 3가지를 제안합니다. Claude가 연동되어 개인 Claude 계정으로 로그인 후 사용할 수 있으며, 사용량은 개인 구독 한도에서 차감됩니다.

문구 추천받기웹 데모 · 예시·규칙 기반 샘플
💡 이 웹 데모의 추천은 예시·규칙 기반 샘플입니다. 서로 다른 접근의 실제 AI 추천 3가지는 플러그인을 설치하고 개인 Claude 계정으로 로그인해야 사용할 수 있어요(사용량은 개인 구독에서 차감).
브라우저 안에서 예시·규칙으로 제안해요(서버 전송 없음)
추천받기를 누르면 여기에 결과가 표시됩니다
설치 방법 — 처음 등록할 때 펼쳐보세요
  1. 받은 ZIP을 압축 해제하면 s1-ux-writing-assistant 폴더가 생깁니다 (안에 manifest.json · code.js · ui.html)
  2. 이 폴더를 영구적인 위치(예: 문서 폴더)로 옮겨 둡니다 — Figma가 이 경로를 참조합니다
  3. Figma Desktop App 상단 메뉴 Plugins → Development → Import plugin from manifest…
  4. 폴더 안의 manifest.json을 선택합니다
  5. 검토할 Figma 파일을 열고 Plugins → Development → S-1 S/W UX Writing Assistant 클릭
  6. 텍스트(또는 프레임)를 선택하고 [검토하기] — 캔버스에 형광펜·코멘트로 검토 결과가 표시됩니다
  7. 문구 추천·대화형 문구 제작은 [문구 추천받기] 탭에서 개인 Claude 로그인 후 사용합니다
폴더를 옮기거나 삭제하지 마세요. Figma는 등록 시점의 폴더 경로를 참조하므로, 폴더가 사라지면 "manifest를 찾을 수 없음" 오류가 납니다. (Downloads 폴더는 자동 정리될 수 있어 권장하지 않습니다.)
