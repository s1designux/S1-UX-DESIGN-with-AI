🔎 검사기 — 실수를 자동으로 막아주는 장치

각각이 무엇을 지키는지, 그리고 저장할 때 자동으로 막아주는지(자동 감시) 보여줍니다.

1 자동 감시
🔎 부품명세 검사기
컴포넌트 설명이 빠짐없이 적혀 있나
2 필요 시 점검
🔎 Figma연결 검사기
피그마 연결(노드)이 살아있나
3 자동 감시
🔎 색상규칙 검사기
색을 규칙대로 썼나 (색코드 직접 입력 금지)
4 자동 감시
🔎 리포트색인 검사기
작업 기록이 목록에 빠짐없이 들어갔나
5 필요 시 점검
🔎 컴포넌트표출 검사기
컴포넌트가 크기별·다크모드별로 제대로 보이나
6 자동 감시
🔎 설치기누락 검사기
설치 파일에 토큰이 하나도 안 빠졌나
7 자동 감시
🔎 토큰값일치 검사기
코드·사이트·설치기가 똑같은 값을 쓰나
8 자동 감시
🔎 부품–변수연결 검사기
컴포넌트가 쓰는 변수가 다 준비됐나
9 자동 감시
🔎 사이즈·숫자페이지 검사기
크기·숫자 값이 페이지에 맞게 나오나
10 자동 감시
🔎 문서토큰이름 검사기
문서에 옛날 토큰 이름이 남아있나
11 자동 감시
🔎 부품해부 검사기
컴포넌트에 꼭 필요한 부품(입력 커서·삭제 아이콘 등)이 들어갔나
12 자동 감시
🔎 아이콘인스턴스 검사기
아이콘을 라이브러리 부품으로 제대로 넣었나
13 자동 감시
🔎 설치기빌드검증 검사기
설치기 코드를 만든 사람 말고 다른 검증을 거쳤나
14 자동 감시
🔎 풋터콘텐츠 검사기
회사 정보·저작권이 진짜 원본과 같나 (지어내기 금지)
15 자동 감시
🔎 토큰네이밍 검사기
토큰 이름을 정한 규칙대로 지었나

🤝 작업 도우미 — 필요할 때 불려나오는 일꾼

총괄이 일을 나눠서 맡기는 도우미들입니다. 만드는 일과 검증하는 일이 서로 나뉘어 있어요.

⭐ 총괄 총괄
river님의 목표를 받아 계획을 세우고, 아래 도우미들을 부르고, 검사기를 돌려 결과를 책임집니다.
🤖 Figma원본 조사
피그마에 있는 진짜 디자인을 읽어옵니다.
피그마 원본을 봐야 할 때
🤖 가이드화면 제작
홈페이지의 가이드 화면을 만들고 고칩니다.
가이드 페이지를 만들거나 고칠 때
🤖 원본대조 검증
만든 결과가 원본과 같은지, 만든 사람과 따로 떨어져 검사합니다.
중요한 변경을 검증할 때
🤖 토큰값 전파
색·크기 값 하나를 바꾸면 코드·사이트·설치기 전부에 빠짐없이 반영합니다.
토큰 값을 바꿀 때
🤖 토큰구조 검사
토큰의 짜임새와 이름이 규칙에 맞는지 봅니다.
토큰 구조를 만들거나 고칠 때
🪞 레거시화면 빌드
옛날 화면을 지금 부품으로 똑같이 다시 그립니다.
옛 화면을 그대로 재현할 때
🏗️ 라이브러리 빌드
피그마 컴포넌트(부품) 자체를 만들고 고칩니다.
피그마 부품을 빌드할 때

⚙️ 자동 장치 — 늘 켜져 있는 안전장치

따로 부르지 않아도 항상 작동하면서 실수를 막습니다.

🚧 커밋 검문소 늘 켜짐
저장하기 직전에 전체를 한 번 검사해서, 틀린 게 있으면 저장을 막습니다.
🔄 토큰편집 동기화기 늘 켜짐
토큰을 고치면 자동으로 다른 곳에도 맞춰줍니다.
⚙️ 피그마 색 차단 늘 켜짐
피그마에 색을 코드로 직접 쓰려 하면 막습니다 (반드시 변수로).
이 화면은 손으로 쓴 게 아니라 자동으로 만들어집니다. 검사기·도우미가 늘거나 바뀌면 목록 파일만 고치고 다시 생성하면 갱신돼요 — 그래서 이 화면 자체는 실제와 어긋나지 않습니다.
