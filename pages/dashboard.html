👥 에이전트 팀 구성

검사기(🔎)가 '정해둔 규칙대로인가'를 기계적으로 자동 점검한다면, 여기 팀원들은 읽고·만들고·판단하는 일을 합니다 — 규칙으로 딱 떨어지지 않아 사람 같은 판단이 필요한 일이라 검사기가 아니라 에이전트입니다. 총괄(⭐)이 위에서 지휘하고, 아래 도우미들은 ① 읽기 → ② 설계 → ③ 만들기 → ④ 검증 순서로 일합니다. 특히 만드는 일(③)과 검증하는 일(④)은 일부러 다른 팀원이 맡습니다(자기 일을 자기가 검사하면 관대해지니까).

🔎 검사기 — 실수를 자동으로 막아주는 장치

비슷한 일을 하는 검사기끼리 6개 묶음으로 정리했습니다. 탭을 눌러 묶음별로 보세요. ("안 걸린다 = 잘 막고 있다"는 뜻이에요.)

컴포넌트를 만들 때 빠진 부품·잘못된 방식이 없는지

Gate 1 자동 감시
🔎 부품명세
컴포넌트 설명이 빠짐없이 적혀 있나
Gate 11 자동 감시
🔎 부품해부
컴포넌트에 꼭 필요한 부품(입력 커서·삭제 아이콘 등)이 들어갔나
Gate 12 자동 감시
🔎 아이콘 인스턴스
아이콘을 라이브러리 부품으로 제대로 넣었나
Gate 13 자동 감시
🔎 설치기 빌드검증
설치기 코드를 만든 사람 말고 다른 검증을 거쳤나

정해둔 값이 코드·사이트·설치기에 똑같이 나오는지

Gate 7 자동 감시
🔎 토큰값 일치
코드·사이트·설치기가 똑같은 값을 쓰나
Gate 9 자동 감시
🔎 사이즈·숫자 페이지
크기·숫자 값이 페이지에 맞게 나오나

필요한 토큰·변수·설치 파일이 빠짐없이 준비됐는지

Gate 6 자동 감시
🔎 설치기 누락
설치 파일에 토큰이 하나도 안 빠졌나
Gate 6b 자동 감시
🔎 설치기 최신
설치 파일이 옛날 버전인 채 남아있지 않나
Gate 8 자동 감시
🔎 부품–변수 연결
컴포넌트가 쓰는 변수가 다 준비됐나

정해둔 표기·이름 규칙을 어긴 게 없는지

Gate 3 자동 감시
🔎 색상 규칙
색을 규칙대로 썼나 (색코드 직접 입력 금지)
Gate 15 자동 감시
🔎 토큰 네이밍
토큰 이름을 정한 규칙대로 지었나
Gate 10 자동 감시
🔎 문서 토큰이름
문서에 옛날 토큰 이름이 남아있나
Gate 17 자동 감시
🔎 미사용 토큰
아무 데서도 안 쓰는 토큰(레거시)이 쌓이나

법인정보·약관 등 검증된 고정 문구가 원본과 같은지

Gate 14 자동 감시
🔎 원본대조 문구
법인정보·약관·브랜드 등 검증된 고정 문구가 원본과 같나 (지어내기 금지)

피그마 연결·화면 표시·기록 색인이 제대로 됐는지

Gate 2 필요 시 점검
🔎 Figma 연결
피그마 연결(노드)이 살아있나
Gate 5 필요 시 점검
🔎 컴포넌트 표출
컴포넌트가 크기별·다크모드별로 제대로 보이나
Gate 4 자동 감시
🔎 리포트 색인
작업 기록이 목록에 빠짐없이 들어갔나
Gate 18 자동 감시
🔎 컴포넌트페이지 커버리지
설치기에 있는 컴포넌트가 HTML 페이지에도 다 있나 (설치기=기준)

⚙️ 자동 장치 — 늘 켜져 있는 안전장치

따로 부르지 않아도 항상 작동하면서 실수를 막습니다.

🚧 커밋 검문소 늘 켜짐
저장하기 직전에 전체를 한 번 검사해서, 틀린 게 있으면 저장을 막습니다.
🔄 토큰편집 동기화기 늘 켜짐
토큰을 고치면 자동으로 다른 곳에도 맞춰줍니다.
⚙️ 피그마 색 차단 늘 켜짐
피그마에 색을 코드로 직접 쓰려 하면 막습니다 (반드시 변수로).
🔤 피그마 폰트 차단 늘 켜짐
피그마에 글자를 넣을 때 정해진 폰트가 아니면 막습니다 (반드시 정본 폰트로).
이 화면은 손으로 쓴 게 아니라 자동으로 만들어집니다. 검사기·도우미가 늘거나 바뀌면 목록 파일만 고치고 다시 생성하면 갱신돼요 — 그래서 이 화면 자체는 실제와 어긋나지 않습니다.
