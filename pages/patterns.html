패턴 레이아웃에서 직접 사용하는 Semantic 토큰입니다. 각 컴포넌트의 Component 토큰은 컴포넌트 가이드를 참조하세요.

용도토큰레이어비고
페이지 배경--color-bg-defaultSemantic전체 페이지 bg
섹션 구분선--color-border-defaultSemantic검색바 하단 구분
결과 카운트 텍스트--color-text-secondarySemantic"N개 결과" 레이블

테이블 행 선택 bg--color-bg-selected (candidate)SemanticFigma 확인 후 stable 전환 예정
  • 1 검색 인풋은 항상 패턴 최상단에 위치한다. 필터 칩은 검색바 바로 아래 행에 배치한다.
  • 2 테이블 헤더는 스크롤 시 고정(sticky)된다. position: sticky; top: 0. z-index는 테이블 헤더 전용 레이어를 사용한다.
  • 3 행 hover 상태와 selected 상태는 반드시 시각적으로 구분된다. hover는 --table-row-hover-bg, selected는 --table-row-selected-bg(→ --color-bg-selected)를 사용한다.
  • 4 페이지네이션은 테이블 하단에 고정 배치한다. 결과 카운트("총 N개")는 페이지네이션 왼쪽에 배치한다.
  • 5 필터 칩을 초기화하는 "전체 초기화" 버튼은 필터 칩 행 우측 끝에 위치한다. secondary 또는 blue-line variant를 사용한다.
  • 6 이 패턴 자체는 드롭다운 + 결과 패널을 포함하지 않는다. 드롭다운 + 결과 패널이 추가되면 Search Module(별도 도메인 패턴)으로 분리한다.
