Variant 선택 기준

Primary
페이지에서 가장 중요한 단일 액션에 사용합니다. 한 화면에 하나만 배치하는 것이 원칙입니다.
Secondary
Primary 버튼과 함께 쓰이는 보조 액션에 사용합니다. '취소', '이전' 같은 역방향 액션에 적합합니다.
Blue-line
배경이 있는 영역에서 Primary 다음 레벨의 강조 액션에 사용합니다. 폼 내 '검색', 필터 적용 등에 적합합니다.
Ghost (deprecated)
배경 없는 텍스트 스타일 액션입니다. V2.4에서 deprecated 확정. 기존 레거시 코드에서만 허용됩니다.
신규 사용 금지

Do / Don't

✓ Do
  • Primary는 화면당 하나만 사용한다
  • 버튼 텍스트는 동사+목적어 형태로 명확하게 작성한다 (예: '저장하기', '파일 업로드')
  • disabled 상태에는 disabled 속성 또는 .is-disabled 클래스를 사용한다
  • 로딩 중에는 .sw-button--loading + aria-busy="true"를 함께 사용한다
  • 아이콘 전용 버튼에는 aria-label을 반드시 지정한다
✕ Don't
  • 같은 영역에 Primary 버튼을 2개 이상 배치하지 않는다
  • 버튼 텍스트를 '클릭', '버튼'처럼 동작을 설명하지 않는 단어로 작성하지 않는다
  • Danger variant를 재추가하지 않는다 (삭제 확정)
  • 인라인 style로 버튼 색상을 직접 override하지 않는다
  • 포커스 ring을 outline: none으로 제거하지 않는다

Accessibility

  • focus-visible — :focus-visible 구현 완료. outline: 2px solid focus-ring token.
  • disabled — pointer-events:none + cursor:not-allowed. disabled 속성 또는 .is-disabled 클래스 모두 처리.
  • loading — pointer-events:none + cursor:wait. aria-busy="true" 추가 권장.
  • keyboard — 기본 button 요소 사용으로 Tab/Enter/Space 지원됨.
  • ARIA — 아이콘만 있는 버튼은 aria-label 필수. 현재 미검증 (a11yStatus: pending).

Review Notes

⚠ Dark Mode 검증 필요

Dark mode 버튼 상태는 semantic token remapping으로 자동 전환됩니다. 그러나 다음 항목은 시각 검증이 필요합니다:

  • --button-primary-disabled-bg: var(--color-border-default) — Dark에서 rgba 값 참조 중 (candidate)
  • Blue-line dark 상태 — 미검증 (darkModeStatus: pending)
