- 파일: 영상출입 통합관제 최종(KOR)_v0.0.6

## 반복/구조 관찰

1. **사이즈 변형 3종이 한 컴포넌트의 반응형 변형**(1×1 / 1×2 / 2×2). 헤더는 동일 `Widget Header` 인스턴스 3곳 공유 → **위젯01과 동일한 core "Widget Header"** 재사용.
2. **핵심 반복 = 지표 3종**(기기 상태 / 이벤트 발생 / 출입문 혼잡도). 1×1·1×2에선 불릿+라벨+뱃지의 리스트 행으로, 2×2에선 아이콘 타일(ic_기기설정/ic_날짜·달력/ic_계정)로 표현 — 같은 3개 지표를 사이즈별로 다르게 표출(관찰).
3. **구역별 점수 표 반복 구조**: 헤더행(구역명/점수/기기상태/이벤트발생/출입문혼잡도) + 데이터행 N개. 각 행 = 5개 `Table` 셀 + 하단 border. 1×2=3행, 2×2=6 placeholder + 1 실데이터.
4. **점수 스택바(3분할 막대)**: 실데이터 행의 `점수` 컬럼에만 등장. 3칸 모두 `green-dark/350`(녹색) + `안전` 라벨 → 점수 구간별 색 변화 예상되나 시안엔 녹색 예시만(관찰).
5. **placeholder 일관**: 대부분 숫자·게이지·셀은 `gray-dark/300`(#2e2f38) 회색 사각형(더미). 실제 데이터는 2×2 표 마지막 행 1건뿐.
6. **아이콘**: 2×2 cards의 3개 아이콘만 표시(ic_기기설정·ic_날짜/달력·ic_계정). 나머지 변형엔 표시 아이콘 없음(불릿 점 Ellipse만).

## 미확인 항목

- **점수 게이지의 실제 형태**: 좌측 큰 회색 사각형(125×62·102×51·224×97)이 반원 게이지/도넛/막대 중 무엇인지 시안엔 없음(placeholder) → 미확인.
- **지표 뱃지·표 셀의 실제 값**: 대부분 placeholder라 실수치 없음. 실데이터는 2×2 표 마지막 행(정문로비/100점/80/90/10.0)만 → 나머지 미확인.
- **점수 스택바 3칸의 의미·구간 색**: 3칸 분할이 무엇을 나눈 것인지(예: 기간/구역/등급), 녹색 외 주의·위험 색이 어떤 값에서 바뀌는지 시안엔 녹색 예시만 → 미확인.
- **cards 타일 노드명 vs 라벨 불일치**: 노드명은 `임직원/내방객/협력사`인데 표시 라벨은 `기기 상태/이벤트 발생/출입문 혼잡도` — 다른 위젯 재활용 흔적으로 보이나 의도 확정은 → 미확인(관찰만).
- **`Default` 외 상태 존재 여부**: 각 변형 위 태그가 `Default`뿐 — hover/loading/error 등 다른 상태 변형은 이 시안에 없음 → 미확인.
- **숨김 툴팁(`알고리즘`)의 표출 조건**: hidden=true 상태라 어떤 인터랙션에서 노출되는지 시안상 미명시 → 미확인.
- **body/primary·secondary·tertiary 색의 최종 라이트/다크 바인딩**: 코드 fallback(#202020/#353535/#757575)은 라이트값, get_variable_defs는 다크 해석(#ecedf0/#b8babf/#8a8c96) — Variable 이름은 확인, 노드별 최종 렌더값은 테마 컨텍스트에 따름.
