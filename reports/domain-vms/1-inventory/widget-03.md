# 위젯 03. 일일 출입 등록자 현황 — 재고조사

- 노드: 1965:15326 / 크기: 2902×1600
- 파일: 영상출입 통합관제 최종(KOR)_v0.0.6
- 스크린샷 관찰: 오늘 새로 등록된 출입 등록자 수를 "신규 등록자" 큰 숫자 + 전일/평균 대비 지표로 요약하고, 임직원/내방객/협력사(외부) 유형별로 나눠 보여주는 관제 대시보드 위젯. 큰 사이즈에는 최근 추이 그래프와 유형별 구성 영역이 추가된다. 같은 위젯의 **3가지 사이즈 변형(1×1 / 1×2 / 2×2)** 이 나란히 배치돼 있고, 별도로 오류 Alert(토스트) 1개가 캔버스 옆에 떨어져 있다. 숫자·그래프 자리는 대부분 회색 placeholder(더미)이며 일부(2×2 신규 등록자 대비)에만 실제 예시 수치가 들어 있다.

> **읽기 방법 표기:** 색은 get_variable_defs로 바인딩된 Variable 이름을 우선 기록(정본). Tailwind 코드의 `#RRGGBB`는 라이트-테마 fallback이라 실제 다크 렌더값과 다를 수 있어 참고로만 병기. 텍스트는 get_design_context 원문 verbatim.

## 반복/구조 관찰

1. **사이즈 변형 3종이 한 컴포넌트의 반응형 변형**(1×1 / 1×2 / 2×2). 헤더는 동일 `Widget Header` 인스턴스를 3곳이 공유.
2. **핵심 반복 = 유형 3종(임직원·내방객·협력사)**. [유형 라벨 + count placeholder + (1×2·2×2) 전일/평균 대비] 구조. 크기가 커질수록 유형별 대비 지표·색 점 범례·막대가 추가됨.
3. **대비 지표 패턴 반복**: `전일 대비 (값)` / `평균 대비 (값)` 한 쌍이 신규 등록자 블록과 각 유형 블록에 반복 등장. 값은 대부분 `(-)` placeholder, 2×2 신규 등록자만 실제 `(44)`/`(37)`.
4. **영역 확장 규칙**: 1×1(신규+유형 3행) → 1×2(+누적 등록, 유형별 막대·대비) → 2×2(+최근 주말 추이 그래프, +유형별 구성 섹션). 큰 변형일수록 섹션이 누적된다.
5. **그래프(2×2)**: 축(Y 100/75/50/25, X 4/18~4/26) + 기준선은 표시, 막대·툴팁·Cursor는 전부 hidden=true → 데이터 없는 골격만 렌더(관찰).
6. **placeholder 일관**: 숫자·막대 자리는 `gray-dark/300`(#2e2f38) 회색 사각형(더미). 위젯01·02와 동일 색.
7. **위젯01·02와 공유**: Widget Header(인스턴스)·Alert(구조)·placeholder 색(gray-dark/300)·Title container 배너 패턴·색 점 범례(ellipse 6×6, 위젯02 1×1과 동일 방식)가 공유됨.
8. **네이밍 이슈(관찰)**: 3변형 내부 프레임명 모두 `Widget 1x1`, 1×1 신규 블록명 `신규 둥록자`(오타), 2×2 그래프 프레임명 `최근 5일 추이` vs 표시 `최근 주말 추이` 불일치 — 레이어명 신뢰도 낮음, 표시 텍스트 우선.

## 미확인 항목

- **숫자·막대·그래프의 실제 데이터**: 대부분 `gray-dark/300` 더미. 실제 수치는 2×2 신규 등록자 대비값(전일 44 / 평균 37)과 그래프 축 눈금(100/75/50/25, 4/18~4/26)뿐 → 그 외 미확인.
- **증감 방향(▲/▼)·색 규칙**: 1×2 내방객의 델타 뱃지(Polygon+`10`)와 2×2 툴팁이 모두 hidden → 상승/하락 색(빨강/파랑 등) 규칙 미확인.
- **외부 유형 라벨·색 점의 Variable 색**: 유형 라벨(임직원 등)과 ellipse 범례 점의 개별 색은 design_context 미조회 → 미확인.
- **Alert 메시지 verbatim**: 위젯03 Alert의 title 텍스트를 직접 재조회하지 않음(구조는 위젯01·02와 동일 확인). 문구 동일 추정이나 verbatim 미확인.
- **Widget Header 내부 타임스탬프/새로고침 verbatim**: 인스턴스 구조만 확인, 위젯03 헤더 내부 design_context 미재조회(위젯01·02와 동일 마스터 추정).
- **body/title 색의 다크 실제값**: 코드 fallback(#202020/#353535/#757575)은 라이트 테마값. variable_defs는 title/primary=#ecedf0, body/secondary=#b8babf, body/tertiary=#8a8c96로 다크 해석 — 노드별 최종 바인딩은 재확인 필요(Variable 이름은 확인됨).
