# 위젯 01. 기기 설치 현황 — 재고조사

- 노드: 1965:14552 / 크기: 2902×1600
- 파일: 영상출입 통합관제 최종(KOR)_v0.0.6
- 스크린샷 관찰: 카메라·서버·컨트롤러·리더 4종 기기의 설치/이상 상태를 집계해 보여주는 관제 대시보드 위젯. 같은 위젯의 **3가지 사이즈 변형(1×1 / 1×2 / 2×2)** 이 나란히 배치돼 있고, 별도로 오류 Alert(토스트) 1개가 캔버스 옆에 떨어져 있다. 숫자·차트 자리는 모두 회색 placeholder 사각형으로 비어 있다(더미 데이터).

> **읽기 방법 표기:** 색은 get_variable_defs로 바인딩된 Variable 이름을 우선 기록(정본). Tailwind 코드의 `#RRGGBB`는 라이트-테마 fallback이라 실제 다크 렌더값과 다를 수 있어 참고로만 병기. 텍스트는 get_design_context 원문 verbatim.

## 반복/구조 관찰

1. **사이즈 변형 3종이 한 컴포넌트의 반응형 변형**(1×1 / 1×2 / 2×2). 헤더는 동일 `Widget Header` 인스턴스를 3곳이 공유 → 공통 core 컴포넌트 후보.
2. **기기 4종(카메라·서버·컨트롤러·리더) 반복 타일**이 핵심 반복 단위. 타일 = [기기명 라벨 + (숨김)아이콘 + 서브카운트 + 합계뱃지 + 차트 placeholder]. 2×2에선 그 아래 **이상상태 리스트(3행)** 가 추가.
3. **리스트 행 반복 구조**: `List` = [상태 라벨(12M) + 우측 count(16×16)] + 행간 divider. 상태별로 라벨 색이 primary/tertiary로 갈림(강조/비활성 구분 관찰).
4. **영역 구분**: ①Title container(위젯 제목 헤더, 최상단 2902폭 배너) ②각 위젯 본체 = 헤더 + 본문 ③기기 열 사이 세로 divider ④Alert는 본체 밖 별도 오류 상태 요소.
5. **placeholder 일관**: 모든 숫자·차트 자리는 `gray-dark/300`(#2e2f38) 회색 사각형(더미). 실제 데이터·차트 종류는 이 시안에 없음.
6. **아이콘 처리**: 기기명 옆 아이콘(ic_박스카메라 등) 4종 모두 hidden=true → 현재 시안에선 미표시. 표시 아이콘은 헤더의 `ic_새로고침`, Alert의 `ic_주의(원형)`·`ic_닫기`뿐.

## 미확인 항목

- **차트 placeholder의 실제 콘텐츠 종류**: 큰 회색 사각형(117×56 등)이 막대/도넛/게이지 등 어떤 차트인지 시안에 없음(더미 placeholder) → 미확인.
- **count/숫자 실제 값**: 서브카운트·합계뱃지·리스트 count 모두 placeholder라 실제 수치 없음(컨트롤러 3행째 `0`만 예외적으로 텍스트 존재) → 미확인.
- **body/primary·tertiary 색의 다크 실제값**: 코드 fallback(#202020/#757575)은 라이트 테마값. get_variable_defs(2×2)는 `body/primary`=#ecedf0, `body/tertiary`=#8a8c96로 다크 해석 → 리스트 개별 행의 라이트/다크 바인딩 최종값은 노드별 재확인 필요(Variable 이름은 확인됨).
- **Alert의 위젯 연동 방식**: Alert가 어느 상태(로딩 실패 등)에서 위젯 본체를 대체/오버레이하는지 시안상 관계 미명시 → 미확인.
- **1×1 변형의 셀 세부 색/구조**: 스크린샷·metadata로 라벨+placeholder 구성은 확인했으나 개별 셀의 design_context는 미조회(2×2 타일 구조로 유추 가능하나 verbatim 확인은 안 함) → 부분 미확인.
