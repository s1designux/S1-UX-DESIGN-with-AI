# 위젯 04. 출입문 대기열 현황 — 재고조사

- 노드: 1965:15542 / 크기: 2902×1600
- 파일: 영상출입 통합관제 최종(KOR)_v0.0.6
- 스크린샷 관찰: 장소(출입문)별 대기열 **혼잡도·대기자수·평균**을 표(테이블) 형태로 보여주는 관제 대시보드 위젯. 같은 위젯의 **3가지 사이즈 변형(1×1 / 1×2 / 2×1)** 이 세로로 배치돼 있고, 별도로 오류 Alert(토스트) 1개가 캔버스 옆에 떨어져 있다. 표의 데이터 행 대부분은 회색 placeholder 막대(더미)이며, 각 변형의 **마지막 행에만 실제 샘플 값**(C동 정문 / 보통 / 12 / 10 등)이 채워져 있다.

> **읽기 방법 표기:** 색은 get_variable_defs로 바인딩된 Variable 이름을 우선 기록(정본). Tailwind 코드의 `#RRGGBB`는 다크-테마 해석값 병기. 텍스트는 get_design_context 원문 verbatim.

## 반복/구조 관찰

1. **사이즈 변형 3종이 한 컴포넌트의 반응형 변형**(1×1 / 1×2 / 2×1). 헤더는 동일 `Widget Header` 인스턴스를 3곳이 공유. 1×1/1×2는 단일 세로 테이블, 2×1은 동일 테이블 2블록을 가로로 나란히(가운데 세로 divider `Line 136`).
2. **핵심 반복 단위 = 데이터 행(row)**. 각 행 = 4개 `Table` 셀(장소명 60 / 혼잡도 flex / 대기자수 46 / 평균 46). 대부분 회색 placeholder 막대, 마지막 행만 실제 샘플 값.
3. **혼잡도 셀 = 텍스트 라벨(보통) + 3분할 막대 게이지**. 게이지는 3세그먼트 중 채운 칸을 `blue-dark/350`(#4285e8), 빈 칸을 `gray-dark/300`(#2e2f38)으로 표현 → 혼잡도 레벨 시각화(관찰). 양끝만 radius 16(알약형).
4. **텍스트 위계**: 컬럼 헤더=Body/10M primary, 장소명·평균=Body/12R secondary(#b8babf), 대기자수=Body/12M primary(#ecedf0 강조), 혼잡도=Body/12M blue 강조.
5. **테이블 라인 규칙**: 헤더 행 하단 = `line/gray/strong`(#35363f, 진함), 데이터 행 하단 = `line/gray/default`(#2e2f38, 옅음), 컬럼 사이 = 세로 divider(90° 회전).
6. **위젯 셸·placeholder·Alert·Title container는 위젯01과 공통**: 위젯 배경 `gray-dark/100`(#1c1d23)+border `line/gray/subtle`+radius 8+padding 16, placeholder 색 `gray-dark/300`(#2e2f38), Alert 토스트 구조, 상단 Title container 배너.

## 미확인 항목

- **Widget Header 변형 관계**: 위젯04 헤더는 타이틀+구분선만 표출(위젯01은 타임스탬프·새로고침 포함). 동일 컴포넌트의 다른 variant인지 별개 컴포넌트인지 Stage 2 판단 필요 → 이름 동일·마스터ID 상이(미확인).
- **Alert 텍스트 verbatim**: 위젯01과 동일 구조로 간주해 이 노드 design_context는 미조회 → 텍스트 원문·색 미확인.
- **placeholder 행의 실제 값**: 대부분 행이 회색 막대(더미)라 실제 장소명·혼잡도·수치 없음. 실제값은 각 변형 마지막 1행(C동 정문/보통/12/10, 출입문 1/보통/10/10)만 존재 → 나머지 미확인.
- **혼잡도 레벨 단계 수·색 매핑**: 샘플은 "보통"(3칸 중 2칸, blue) 1케이스만. 다른 레벨(혼잡/여유 등)의 색·칸 수 매핑은 시안에 없음 → 미확인.
- **시간대 툴팁 트리거**: `Group 2085672892`(출근 09:00~10:30 툴팁)가 hidden 상태. 어떤 인터랙션(hover 등)에서 표출되는지 시안상 미명시 → 미확인.
