# Modal (공통 팝업) 규격 문서

> 작성 2026-07-03 · 상태: **확정(HD-1~4 전건 결정)** · 근거: 레거시 원본 3세트 분석(파일 `P8YvnCdGkQLDNVQhW74ZZW`) + 실측 토큰(get_variable_defs)
> 목적: 흩어져 있는 모달 예시를 **하나의 규격**으로 유형화해, 만들기 전에 기준을 확정한다. (레거시는 "정답"이 아니라 출발점 — 색·수치는 우리 정본 토큰으로 매핑)

## 1. 한 줄 요약

모달은 **"헤더 + 본문 + 푸터"의 3층 껍데기 하나**로 정의한다.
- 껍데기(헤더·푸터)는 **고정**, 본문만 **콘텐츠에 따라 갈아끼운다**.
- 변형(variant)은 **크기 4단계**와 **푸터 버튼 1개/2개** 두 축만 둔다.
- 본문은 새로 만들지 않고 **이미 있는 코어 컴포넌트(버튼·인풋·셀렉트·날짜선택·표 등)를 조합**한다.

---

## 2. 공통 구조 (Anatomy)

모든 모달은 유형과 무관하게 아래 3층을 가진다. 배경에는 딤(overlay)이 깔린다.

```
┌─────────────────────────────────────┐  ← 딤(overlay) 위에 뜸
│  [제목]                        [ ✕ ] │  ← 헤더 (고정)
├─────────────────────────────────────┤
│                                     │
│   본문 (내용 — 유형별로 다름)          │  ← 본문 (내용에 맞춰 늘되, 최대 높이 넘으면 스크롤)
│                                     │
├─────────────────────────────────────┤
│                     [버튼] [버튼]     │  ← 푸터 (고정, 버튼 우측정렬 1~2개)
└─────────────────────────────────────┘
```

| 층 | 구성 | 규칙 |
|----|------|------|
| **헤더** | 제목(왼쪽) + 닫기 아이콘 ✕(오른쪽) | 높이 고정. 제목은 1줄 기준. |
| **본문** | 텍스트·리스트·폼·표·미디어 등 (§5) | 내용에 맞춰 늘되 §3의 세로 규칙 적용. 좌우·상하 패딩 고정. |
| **푸터** | 버튼 1개 또는 2개, 오른쪽 정렬 | 높이 고정. 버튼 사이 간격 고정(8px). |

**공통 패딩·모서리·간격 (레거시 실측 6443:141123 → 정본 매핑)**

| 항목 | 레거시 실측 | 정본 토큰 |
|------|-----------|----------|
| 모서리 반경 | `radius/8` = 8 | `--radius-modal-md` |
| 좌우 안쪽 여백 | `spacing/padding/inline/lg` = 24 | `spacing/24` |
| 상하 안쪽 여백 | `spacing/padding/block/md` = 20 | `spacing/20` |
| **세로 간격(헤더↔본문↔푸터)** | **`spacing/section/lg` = 32** | `spacing/32` ← ⚠️ 16 아님 |
| 본문 줄 간격 | `spacing/8` = 8 | `spacing/8` |
| 푸터 버튼 간격 | `spacing/cluster/xxs` = 8 | `spacing/8` |
| 표면(배경) Light/Dark | #fff / #1c1d23 | **`color/modal/bg`** (신설, light=base/white · dark=gray-dark/100 — chip/button 과 동일 패턴) |
| 딤(overlay) | (반투명 검정) | `color/overlay` (rgba 예외) |
| 그림자 Dark | (참고 781:32832) 드롭섀도우 2겹 | 아래 §7 |

> **레거시 내부 구조(6443:141123):** frame `min-h 184` · `py 20` · 헤더/본문/푸터 세로 gap **32(section/lg)** · 각 섹션 좌우 px 24 · 본문 줄 gap 8 · 푸터 gap 8·우측정렬. (초판에서 gap을 16, min-h를 140으로 잘못 잡음 → 2026-07-03 원본 재정독 후 교정)

## 3. 크기 규격 (핵심)

### 3-1. 크기 4단계 — 폭 × 높이 (✅ 사용자 재정의 2026-07-03)

| 단계 | 폭 | 권장/최소 높이 | 쓰임 | 대표 예시 |
|------|----|--------------|------|----------|
| **S (최소)** | **360** | **174** | 짧은 안내·확인·오류 | 필수항목 미입력(6706:4219) |
| **M** | **520** | **336** | 선택지·간단한 폼 | 다운로드 옵션(540:5826) |
| **L** | **1000** | **587** | 복잡한 정보·업무 화면 | 지정 노선 추가(540:5837) |
| **XL (최대)** | **1200** | **587** | 넓은 표·다중 컬럼 | 대용량 데이터(540:5849) |

> 폭은 4개 값 중 하나로 고정(임의 폭 금지). 높이는 위 값을 **권장(최소)높이**로 — 넓은 모달도 이 높이 유지(내용 적어도), 내용 많으면 그 이상(§3-2). (M 폭 초판 440→520 정정.)

### 3-2. 세로 높이 — 규칙 (✅ 재정의 2026-07-03)

**크기별 권장(최소)높이를 유지하고, 내용이 많으면 그 이상으로 늘어난다.**

1. **각 크기 = 권장(최소)높이** (§3-1: S 174 / M 336 / L 587 / XL 587). 넓은 모달도 내용이 적어도 이 높이를 유지(빈 세로 방지 ← R4 순수hug의 과소 교정).
2. **푸터는 바닥 고정, 헤더·본문은 위에 스냅.** 본문(텍스트 영역)은 자체로 늘어나 공간을 미리 차지하지 않는다(R3 body-fill 오류 금지). 최소높이와 내용 사이 여백은 **space-between**(헤더+본문 그룹 ↕ 푸터)으로 흡수 — body 박스는 스냅 유지.
3. **내용이 권장높이보다 많으면** 프레임이 그 이상으로 hug 하며 늘어난다.
4. **상한 = 85vh**(화면 세로 85%, 딤이 GNB 포함 전체 덮음). 초과 시 헤더·푸터 고정, 본문만 세로 스크롤.

| 항목 | 값 |
|------|-----|
| 권장(최소)높이 | S 174 / M 336 / L 587 / XL 587 (프레임 minHeight) |
| 높이 성장 | 내용 > 최소면 hug 로 늘어남 |
| 푸터 위치 | 바닥 고정(space-between). 본문 스냅(body layoutGrow 금지) |
| 최대높이 | **85vh** (1080 환산 ≈918px), 초과 시 본문 스크롤 |

> **경위:** flat 184(R1) → 크기별 587+body fill(R3, body가 공간 차지=오류) → 순수 hug(R4, 넓은 모달이 내용 적을 때 너무 낮음=과소) → **크기별 권장높이 minHeight + 푸터 바닥(space-between)·본문 스냅**(2026-07-03 사용자 재정의: S 360×174·M 520×336·L 1000×587·XL 1200×587). R3와 달리 body 를 stretch 하지 않고 space-between 으로 여백 흡수.

## 4. 푸터 버튼 규격

버튼 개수 = 모달의 **목적**을 나타낸다. (버튼은 새로 만들지 않고 **코어 Button 컴포넌트 재사용**)

| 유형 | 버튼 | 목적 | 예시 | 오른쪽 끝 버튼 |
|------|------|------|------|--------------|
| **안내형** | 1개 | 알림·완료·오류를 알리고 닫기 | "인증이 완료되었어요 / **확인**" | Primary(파랑) |
| **확인형** | 2개 | 되돌리기 어려운 결정·작업 실행 | "정말 삭제? / 아니오·**네**", "취소·**저장**" | Primary(파랑) |

**규칙**
- 버튼은 **오른쪽 정렬**, 2개일 때 **[보조(취소/아니오)] [주(네/저장/확인)]** 순서, 주 버튼이 오른쪽.
- 버튼 사이 간격 = `spacing/cluster/xxs` = 8px → `--spacing-cluster-xxs`.
- 버튼 색: 주=Button primary(`--color-action-primary-*`), 보조=Button secondary(흰 배경+회색 테두리). (Button 토큰 그대로 재사용)
- **푸터 버튼 크기**: ✅ **모든 크기 모달에서 h34(xsmall)로 통일**(HD-2 확정 2026-07-03). 레거시 작은 모달의 h28은 h34로 통일.

## 5. 본문 콘텐츠 유형 (슬롯)

본문은 껍데기가 스타일을 새로 정하지 않고, 아래 조각들을 **끼워 넣어 조합**한다. 각 조각은 **코어 컴포넌트 재사용**이 원칙.

| 콘텐츠 유형 | 설명 | 예시 | 재사용 코어 |
|------------|------|------|-----------|
| 한 줄 텍스트 | 짧은 안내 | "필수 항목을 입력해주세요." | (텍스트 스타일) |
| 여러 줄 텍스트 | 상황 설명+질문 | "변경 내용이 저장되지 않습니다. 중단할까요?" | (텍스트 스타일) |
| 불릿 리스트 | 규칙·조건 안내 | 비밀번호 조건 | (리스트 스타일) |
| 강조 텍스트 | 숫자·링크 강조 | "총 **000개** 항목", "25.12.30 최종등록" | `--color-text-state-accent`(#1d6ceb), 링크 |
| 폼 컨트롤 | 입력·선택 | 라디오/인풋/셀렉트/날짜·시간 선택 | Input · Select · Radio · **DatePicker** · TimePicker |
| 데이터 표 | 조회 결과 | 운행 정보 조회 | Table |
| 미디어 | 지도·이미지 | 노선 지도 | (이미지 컨테이너) |
| 조합 | 위를 섞음 | 큰 모달 = 지도+폼+표 | 여러 코어 조합 |

**본문 텍스트 토큰 (실측 → 정본)**

| 항목 | 레거시 실측 | 정본 토큰(후보) |
|------|-----------|----------------|
| 제목 | `title/16B` (Pretendard Bold 16) | 색 ✅ **`--color-text-title-primary`**(HD-3) · **타이포=정본 텍스트스타일 `Title/16B`(대문자, 라이브러리 1415:4 — 참고 팝업과 동일)**. 로컬 소문자 `title/16B` 금지 |
| 본문 | `body/14R` (Regular 14), `color/text/body/primary` #353535 | `--color-text-body-primary` 계열(#353535) |
| 닫기 아이콘 | `color/icon/gray-dark` #353535, `sizing/icon/24` | `--color-icon-default` · 24×24 |
| 강조/링크 | `color/text/state/accent` #1d6ceb | `--color-action-primary-default` 계열 |

> ✅ 제목 색: 정본 토큰 **`--color-text-title-primary`**(gray/900, 다크=gray-dark/900) 사용 확정(HD-3, 2026-07-03). 레거시 순검정(#000)은 이 토큰으로 대체.

|------|------|

## 7. 다크모드 (참고 노드 781:32832 기준)

- 색은 전부 Semantic 토큰 경유이므로 **값만 다크로 전환**(표면·텍스트·테두리·버튼·딤). 구조 변경 없음.
- **패널 배경(현재/interim)** = `color/modal/bg`(VariableID:809:2, light=base/white · dark=gray-dark/100 #1c1d23). Figma 변수로 신설·바인딩 완료.
- ⚠️ **다크 표면 토큰은 다른 세션에서 신설 `surface/raised` 로 통일 예정(사용자 결정 2026-07-03).** 모달의 "한 단계 띄운" 표면(카드/모달 raised)을 공통 `surface/raised` 로 묶는다. → **이 세션에서는 color/modal/bg 를 코드(vars-data/tokens.css 등)로 전파하지 않는다**(surface/raised 로 대체될 예정). Figma color/modal/bg 는 interim. 다크 표면 마무리 = 별도 세션.
- **그림자(Dark)** = 참고 781:32832 의 드롭섀도우 2겹(`0 20 12` + `0 8 4`, 어두운 검정 계열)을 그대로 복사. (shadow rgba 예외 허용)
- **서브틀 보더(Dark)** = `color/line/gray`(subtle) — 어두운 배경에서 패널 경계.
- **푸터 버튼**: 다크에서 secondary(취소) = `color/button/bg/secondary--default` 다크값 `gray-dark/100`, 라벨 `gray-dark/800`. → **다크 스펙 프레임은 버튼 토큰 컬렉션까지 Dark 모드를 적용**해야 흰색으로 새지 않음(초판 버그).
- Light 모달은 그림자·보더 없음(레거시 삭제 모달 기준, 딤 위에 플랫).

## 8. 결정 이력 (Human Decision — 전건 확정)

- ✅ **HD-1 · 세로 높이**: 고정 높이 없음 + **최대 85vh**, 초과 시 본문만 스크롤(헤더·푸터 고정). 딤이 GNB 포함 전체를 덮음(경우 1). 픽셀 캡 없음. (확정 2026-07-03)
- ✅ **HD-2 · 푸터 버튼 크기**: **h34(xsmall)로 전 단계 통일** (확정 2026-07-03)
- ✅ **HD-3 · 제목 색**: 정본 **`--color-text-title-primary`**(gray/900) 사용 (확정 2026-07-03)
- ✅ **HD-4 · 닫기 방식**: **✕·버튼으로만** 닫힘. 딤 클릭·ESC 불가 (확정 2026-07-03)
