# Modal (공통 팝업) 규격 문서 — 초안

> 작성 2026-07-03 · 상태: **초안(검토 대기)** · 근거: 레거시 원본 3세트 분석(파일 `P8YvnCdGkQLDNVQhW74ZZW`) + 실측 토큰(get_variable_defs)

## 1. 한 줄 요약

모달은 **"헤더 + 본문 + 푸터"의 3층 껍데기 하나**로 정의한다.
- 껍데기(헤더·푸터)는 **고정**, 본문만 **콘텐츠에 따라 갈아끼운다**.
- 변형(variant)은 **크기 4단계**와 **푸터 버튼 1개/2개** 두 축만 둔다.
- 본문은 새로 만들지 않고 **이미 있는 코어 컴포넌트(버튼·인풋·셀렉트·날짜선택·표 등)를 조합**한다.

---

## 2. 공통 구조 (Anatomy)

모든 모달은 유형과 무관하게 아래 3층을 가진다. 배경에는 딤(overlay)이 깔린다.

```
┌─────────────────────────────────────┐  ← 딤(overlay) 위에 뜸
│  [제목]                        [ ✕ ] │  ← 헤더 (고정)
├─────────────────────────────────────┤
│                                     │
│   본문 (내용 — 유형별로 다름)          │  ← 본문 (내용에 맞춰 늘되, 최대 높이 넘으면 스크롤)
│                                     │
├─────────────────────────────────────┤
│                     [버튼] [버튼]     │  ← 푸터 (고정, 버튼 우측정렬 1~2개)
└─────────────────────────────────────┘
```

| 층 | 구성 | 규칙 |
|----|------|------|
| **헤더** | 제목(왼쪽) + 닫기 아이콘 ✕(오른쪽) | 높이 고정. 제목은 1줄 기준. |
| **본문** | 텍스트·리스트·폼·표·미디어 등 (§5) | 내용에 맞춰 늘되 §3의 세로 규칙 적용. 좌우·상하 패딩 고정. |
| **푸터** | 버튼 1개 또는 2개, 오른쪽 정렬 | 높이 고정. 버튼 사이 간격 고정(8px). |

**공통 패딩·모서리 (실측 → 정본 매핑)**

| 항목 | 레거시 실측 | 정본 토큰(후보) |
|------|-----------|----------------|
| 모서리 반경 | `radius/8` = 8 | `--radius-modal-md` |
| 좌우 안쪽 여백 | `spacing/padding/inline/lg` = 24 | `--spacing-padding-inline-lg` |
| 상하 안쪽 여백 | `spacing/padding/block/md` = 20 | `--spacing-padding-block-md` |
| 표면(배경) | `surface/neutral/bg/base` = #fff | `--color-surface-default` |
| 딤(overlay) | (반투명 검정) | `--color-overlay-*` (rgba 허용 예외) |

## 3. 크기 규격 (핵심)

### 3-1. 가로 폭 — 4단계 (확정)

| 단계 | 폭 | 쓰임 | 대표 예시 |
|------|----|------|----------|
| **S (작은)** | **360px** | 짧은 안내·확인·오류 | 필수항목 미입력, 삭제 확인 |
| **M (중간)** | **440px** | 선택지·간단한 폼 | 다운로드 옵션(라디오+간단폼) |
| **L (큰)** | **1000px** | 복잡한 정보·업무 화면 | 지정 노선 추가(지도+폼+표, 원본 991→1000) |
| **XL (가장 큰)** | **1200px** | 넓은 표·다중 컬럼 | (대용량 데이터·비교 화면) |

> 폭은 **위 4개 값 중 하나로 고정**한다. 임의 폭 금지.

### 3-2. 세로 높이 — 일반 기준(업계 관행) 제안 [HD-1 판단 대기]

**업계 표준(Material·Ant·Carbon 등)은 모달에 고정 높이를 두지 않는다.** 대신 아래 방식으로 "너무 자유롭지 않게" 제한한다.

1. **고정 높이 없음** — 내용 높이에 맞춘다(hug). 짧은 모달은 짧게, 긴 모달은 필요한 만큼만.
2. **상한만 둔다** — 모달 전체 최대 높이 = 화면(뷰포트) 높이 − 상하 여백. 실무 관행상 **최대 ≈ 화면의 85~90%**.
3. 상한을 넘으면 → **헤더·푸터는 고정, 본문만 세로 스크롤**.
4. **하한** — 헤더 + 본문 최소 1줄 + 푸터가 들어갈 최소치.

**권장값 (river 판단용)**

| 항목 | 권장 | 설명 |
|------|------|------|
| 최대 높이 | `calc(100vh − 80px)` (상하 각 40px 여백, ≈ 90vh) | 화면을 꽉 채우지 않고 여백 확보 |
| 기준 해상도(1920×1080) 환산 | 최대 ≈ **1000px** | 관리자 화면 기준 실제 상한 |
| 크기 단계별 픽셀 상한 | **두지 않음** | 폭만 4단계, 높이는 공통 규칙(뷰포트 기반). 내용 짧으면 자연히 짧아짐 |
| 하한 | 헤더+1줄+푸터 (S 기준 약 140px) | 버튼만 덜렁 붙는 것 방지 |

> **왜 픽셀 캡(예: L=720)이 아니라 뷰포트 기준(90vh)인가:** 픽셀 캡은 큰 화면에선 답답하고 작은 화면에선 넘칠 수 있음. 뷰포트 기준은 어느 해상도에서도 "화면을 넘지 않으면서 필요한 만큼" 유지돼 관리가 쉬움. → **이 방식 채택 여부가 HD-1.**

## 4. 푸터 버튼 규격

버튼 개수 = 모달의 **목적**을 나타낸다. (버튼은 새로 만들지 않고 **코어 Button 컴포넌트 재사용**)

| 유형 | 버튼 | 목적 | 예시 | 오른쪽 끝 버튼 |
|------|------|------|------|--------------|
| **안내형** | 1개 | 알림·완료·오류를 알리고 닫기 | "인증이 완료되었어요 / **확인**" | Primary(파랑) |
| **확인형** | 2개 | 되돌리기 어려운 결정·작업 실행 | "정말 삭제? / 아니오·**네**", "취소·**저장**" | Primary(파랑) |

**규칙**
- 버튼은 **오른쪽 정렬**, 2개일 때 **[보조(취소/아니오)] [주(네/저장/확인)]** 순서, 주 버튼이 오른쪽.
- 버튼 사이 간격 = `spacing/cluster/xxs` = 8px → `--spacing-cluster-xxs`.
- 버튼 색: 주=Button primary(`--color-action-primary-*`), 보조=Button secondary(흰 배경+회색 테두리). (Button 토큰 그대로 재사용)
- **푸터 버튼 크기**: ✅ **모든 크기 모달에서 h34(xsmall)로 통일**(HD-2 확정 2026-07-03). 레거시 작은 모달의 h28은 h34로 통일.

## 5. 본문 콘텐츠 유형 (슬롯)

본문은 껍데기가 스타일을 새로 정하지 않고, 아래 조각들을 **끼워 넣어 조합**한다. 각 조각은 **코어 컴포넌트 재사용**이 원칙.

| 콘텐츠 유형 | 설명 | 예시 | 재사용 코어 |
|------------|------|------|-----------|
| 한 줄 텍스트 | 짧은 안내 | "필수 항목을 입력해주세요." | (텍스트 스타일) |
| 여러 줄 텍스트 | 상황 설명+질문 | "변경 내용이 저장되지 않습니다. 중단할까요?" | (텍스트 스타일) |
| 불릿 리스트 | 규칙·조건 안내 | 비밀번호 조건 | (리스트 스타일) |
| 강조 텍스트 | 숫자·링크 강조 | "총 **000개** 항목", "25.12.30 최종등록" | `--color-text-state-accent`(#1d6ceb), 링크 |
| 폼 컨트롤 | 입력·선택 | 라디오/인풋/셀렉트/날짜·시간 선택 | Input · Select · Radio · **DatePicker** · TimePicker |
| 데이터 표 | 조회 결과 | 운행 정보 조회 | Table |
| 미디어 | 지도·이미지 | 노선 지도 | (이미지 컨테이너) |
| 조합 | 위를 섞음 | 큰 모달 = 지도+폼+표 | 여러 코어 조합 |

**본문 텍스트 토큰 (실측 → 정본)**

| 항목 | 레거시 실측 | 정본 토큰(후보) |
|------|-----------|----------------|
| 제목 | `title/16B` (Pretendard Bold 16) | ✅ **`--color-text-title-primary`** (= gray/900, HD-3 확정) · 타이포 title 16B |
| 본문 | `body/14R` (Regular 14), `color/text/body/primary` #353535 | `--color-text-body-primary` 계열(#353535) |
| 닫기 아이콘 | `color/icon/gray-dark` #353535, `sizing/icon/24` | `--color-icon-default` · 24×24 |
| 강조/링크 | `color/text/state/accent` #1d6ceb | `--color-action-primary-default` 계열 |

> ✅ 제목 색: 정본 토큰 **`--color-text-title-primary`**(gray/900, 다크=gray-dark/900) 사용 확정(HD-3, 2026-07-03). 레거시 순검정(#000)은 이 토큰으로 대체.

## 6. 상태 · 동작

| 항목 | 규칙 |
|------|------|
| 열기 | 딤(overlay) + 모달 중앙 표시 |
| 닫기 | ✅ **헤더 ✕ 또는 버튼(취소·아니오)으로만**. 딤(배경) 클릭·ESC로는 닫히지 않음(HD-4 확정, 실수 방지) |
| 스크롤 | 본문 초과분만 세로 스크롤(§3-2). 헤더·푸터 고정 |
| 포커스 | 열리면 모달 안으로 포커스 이동, 뒤 배경 비활성 (접근성) |

## 8. 미결정 (Human Decision)

- **HD-1 · 세로 높이 [판단 대기]**: §3-2 일반 기준(고정높이 없음 + 최대 `calc(100vh−80px)`≈90vh, 초과 시 본문 스크롤, 픽셀 캡 없음)을 채택할까요? 이 방식을 보고 판단 요청받음.
- ✅ **HD-2 · 푸터 버튼 크기**: **h34(xsmall)로 전 단계 통일** (확정 2026-07-03)
- ✅ **HD-3 · 제목 색**: 정본 **`--color-text-title-primary`**(gray/900) 사용 (확정 2026-07-03)
- ✅ **HD-4 · 닫기 방식**: **✕·버튼으로만** 닫힘. 딤 클릭·ESC 불가 (확정 2026-07-03)
